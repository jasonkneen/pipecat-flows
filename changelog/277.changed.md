- Migrated to Pipecat 1.3.0's worker model. `FlowManager` now accepts a `worker` (`PipelineWorker`) argument and exposes a `worker` property. The `task` argument and the `FlowManager.task` property are deprecated but continue to work (a `PipelineTask` is now a `PipelineWorker`), so existing code is unaffected. Examples were updated to use `PipelineWorker` and `WorkerRunner` with `runner.add_workers(worker)` + `runner.run()` (#277).
