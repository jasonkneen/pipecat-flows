- ⚠️ `FlowsFunctionSchema.handler` is no longer typed as optional. In practice, it wasn't optional anyway: the optionality was a holdover from the removed `transition_to`/`transition_callback` fields, which once let a function transition without a handler. Transitions now happen via a handler that returns `(result, next_node)`. Constructing a `FlowsFunctionSchema` without a `handler` now raises an error.
